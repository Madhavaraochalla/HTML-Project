Key Responsibilities:

1. Jenkins Configuration:   
   ➔ Configured Jenkins to automate the process of pulling updates from a GitHub repository. This setup used Maven as the build tool to compile the Java application, ensuring seamless integration of code changes.
2. Pipeline Creation:
   ➔ Designed and implemented Jenkins pipelines that automate the build process. These pipelines included steps to build application artifacts and create custom Docker images using Dockerfiles, ensuring that the build process was both repeatable and reliable.
3. Docker Image Management:
   ➔ Automated the upload process of Docker images to Docker Hub, facilitating version control and distribution. This automation ensured that the latest application versions were always available for deployment.
4. Ansible Deployment:
   ➔ Crafted Ansible playbooks for deploying Docker images across multiple servers. This approach guaranteed consistent application deployment, eliminating configuration drift and ensuring that all servers were running identical versions of the application.
5. Container Orchestration:
   ➔ Implemented Ansible roles on the Ansible server to manage container orchestration tasks. These roles automated the deployment process, allowing for efficient and scalable management of containerized applications.
6. Automated Testing:
   ➔ Integrated Selenium tests within the Jenkins pipeline to automatically validate the functionality of the application post-deployment. This ensured that any issues were detected and addressed promptly, maintaining the integrity and reliability of the application.
7. Kubernetes Management:
   ➔ Utilized Kubernetes to manage the containerized applications. Created deployment manifests and service definitions to ensure high availability, load balancing, and efficient resource utilization. This setup enabled the application to scale seamlessly in response to varying load demands.

Achievements:

2. Enhanced Release Reliability:

3. Scalability and Consistency:
